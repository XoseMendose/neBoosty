# neBoosty

Учебный pet-проект — клон платформы Boosty, разработанный в рамках подготовки к стажировке **«Импульс»**.

Boosty — одна из крупнейших русскоязычных платформ монетизации контента, где авторы публикуют посты, статьи и медиа, а подписчики платят за доступ к материалам. Задача проекта — воспроизвести ключевую функциональность подобной платформы: от авторизации и публикации контента до обработки платежей и системы уведомлений.

---

## Зачем этот проект

Проект создан не ради воспроизведения конкретного продукта, а ради отработки реальных менеджерских задач: самостоятельного проектирования архитектуры, определения границ ответственности между сервисами, декомпозиции продукта на эпики и фичи.

---

## Для кого

Проект — это демонстрация моих практических навыков для команды (для Вас!), рассматривающей мою кандидатуру на стажировку. Он показывает умение самостоятельно спроектировать нетривиальную систему, декомпозировать её на сервисы с понятными границами ответственности и оформить бэклог продукта.

---

## Что внутри

Платформа покрывает полный цикл взаимодействия автора и подписчика: регистрация и аутентификация, публикация и монетизация контента, подписки и платежи, персонализированная лента, поиск, чат, уведомления через email, SMS, Telegram и Discord, а также аналитика и мониторинг всей системы.

## Архитектура 
![Архитектура](https://github.com/XoseMendose/neBoosty/blob/main/%D0%90%D1%80%D1%85%D0%B8%D1%82%D0%B5%D0%BA%D1%82%D1%83%D1%80%D0%B0%20%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F.png)

Проект построен на микросервисной архитектуре. Все входящие запросы проходят через **API Gateway**, который маршрутизирует их по сервисам:
| Сервис | Ответственность |
|---|---|
| Authentication Service | Регистрация и авторизация |
| User Service | Управление профилями подписчиков |
| Author Service | Управление профилями авторов |
| Content Service | Публикация и хранение контента |
| Feed Service | Персональная лента |
| Search Service | Поиск контента |
| Subscription Service | Оформление подписок |
| Payment Service | Платежи и монетизация |
| Social Service | Комментарии и реакции |
| Chat Service | Чат с автором |
| Notification Service | Уведомления |
| Analytics Service | Аналитика |
| Connection Service | Интеграция с Telegram и Discord |

Межсервисное взаимодействие — через **Kafka**. Фоновые задачи (рассылки) — через **Celery**.

## Дорожная карта проекта

Общая разработка рассчитана на 4 месяца. MVP планируется предоставить через 3 месяца с начала разработки: сервис рассылки, аналитики и мониторинга, социальных функий будут введены после разработки MVP. Построение общей архитектуры и подключение к Kafka требуются для подключения микросервисов на всех этапах разработки, поэтому им выделен срок до конца общей разработки.
